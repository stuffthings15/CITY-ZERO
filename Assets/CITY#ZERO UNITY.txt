# CITY//ZERO — Unity Production Package

## 1. Unity Starter Project Structure

### Recommended Unity Version

* Unity 6 LTS preferred for production stability.
* URP recommended for performance, clarity, and easier stylized top-down lighting.
* Input System package required.
* Cinemachine recommended.
* Addressables required.
* Unity AI Navigation package required.
* Entities/DOTS optional for late-stage traffic and crowd scaling, but do not make MVP dependent on DOTS.

### Core Packages

* Input System
* Cinemachine
* AI Navigation
* Addressables
* Timeline
* TextMeshPro
* URP
* Unity Test Framework
* ProBuilder for blockout
* Splines optional for roads and patrols

### Scene Layout Strategy

* `Bootstrap.unity`: initializes services, save profile, settings, addressables.
* `MainMenu.unity`: title, profile selection, options, accessibility.
* `Sandbox_Global.unity`: persistent systems scene loaded additively.
* District scenes loaded additively by streaming volumes.
* Separate test scenes for combat, traffic, AI, missions, and save/load regression.

### Assembly Definition Strategy

Create asmdefs to reduce compile time:

* `CityZero.Core`
* `CityZero.Gameplay`
* `CityZero.UI`
* `CityZero.Data`
* `CityZero.Editor`
* `CityZero.Tests.EditMode`
* `CityZero.Tests.PlayMode`

### Naming Conventions

* Classes: PascalCase
* Private fields: `_camelCase`
* Interfaces: `IName`
* ScriptableObjects: suffix `Config`
* Runtime DTOs: suffix `Data`
* Prefabs: `PF_`
* Materials: `MAT_`
* Audio clips: `SFX_`, `MUS_`, `VO_`
* Scenes: `SCN_`
* Animation controllers: `AC_`

### Unity Architecture Approach

Use hybrid architecture:

* ScriptableObjects for design-time static content
* JSON for bulk tunable content and external editing
* MonoBehaviour gameplay layer for MVP
* Job System or DOTS reserved for later scaling of traffic and crowds
